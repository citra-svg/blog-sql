Memahami Stored Procedure pada Database AKADEMIK

Pada praktikum minggu ini saya mempelajari tentang Stored Procedure pada database. Stored Procedure adalah kumpulan query SQL yang disimpan di dalam database dan dapat dipanggil kembali menggunakan nama procedure. Dengan menggunakan Stored Procedure, query menjadi lebih rapi, mudah digunakan kembali, dan proses pengolahan data menjadi lebih efisien.

Pada praktikum ini digunakan database AKADEMIK yang memiliki beberapa tabel seperti mahasiswa, prodi, dosen, jenjang, dan status_akademik. Saya belajar membuat procedure tanpa parameter dan procedure dengan parameter IN. Selain itu saya juga mempelajari penggunaan ORDER BY, LIMIT, fungsi agregat COUNT, dan JOIN.

Contoh Query 1

Query berikut digunakan untuk menampilkan data program studi dari tabel prodi yang diurutkan berdasarkan nama prodi. 


CREATE PROCEDURE tampilNamaProdi()
BEGIN
  SELECT idprodi, nmprodi
  FROM prodi
  ORDER BY nmprodi;
END;

CALL tampilNamaProdi();

Dari hasil output, data program studi berhasil ditampilkan sesuai urutan alfabet. Procedure tanpa parameter ini mempermudah pengguna untuk melihat seluruh data prodi.

Contoh Query 2

Query berikut digunakan untuk menampilkan data dosen berdasarkan program studi dan pendidikan terakhir dosen. 


CREATE PROCEDURE getDosenByProdiPendidikan(
  IN p_idprodi INT,
  IN p_idpendidikan INT
)
BEGIN
  SELECT nmdosen, gelar
  FROM dosen
  WHERE idprodi = p_idprodi
    AND idpendidikan = p_idpendidikan
  ORDER BY nmdosen;
END;

CALL getDosenByProdiPendidikan(1,2);

Melalui query tersebut saya memahami bahwa parameter dapat digunakan untuk memfilter data sesuai kebutuhan pengguna. Procedure ini mempermudah pencarian data dosen berdasarkan kondisi tertentu.

Refleksi Praktikum

Hal yang cukup sulit saat praktikum adalah memahami perbedaan sintaks antara simulator SQL.js dan SQL Server. Pada simulator digunakan perintah CALL, sedangkan pada SQL Server menggunakan EXEC. Selain itu, beberapa sintaks seperti LIMIT juga perlu disesuaikan agar dapat berjalan dengan baik di simulator.

Tips yang membantu saya selama praktikum adalah memahami struktur tabel terlebih dahulu sebelum membuat procedure. Dengan memahami relasi antar tabel, proses pembuatan query menjadi lebih mudah dan mengurangi kesalahan saat menjalankan procedure.

Kesimpulan

Dari praktikum ini saya dapat memahami bahwa Stored Procedure sangat berguna untuk menyimpan query yang sering digunakan sehingga pengolahan data menjadi lebih efektif, terstruktur, dan mudah dipanggil kembali kapan saja.
